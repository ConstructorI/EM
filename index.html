О проекте

Equipment master - это надежное, масштабируемое и удобное приложение для управления оборудованием, разработанное с использованием современных технологий и лучших практик программирования.

Проект реализован на базе фреймворка Qt с применением модульной архитектуры, объектно-ориентированного подхода и событийно-ориентированного программирования.

Архитектурные решения

Модульность

Каждый модуль отвечает за определенную функциональность: database.cpp - работа с БД, mainwindow.cpp - основной интерфейс, forms.cpp - специализированные формы.

ООП подход

Использование наследования и полиморфизма для создания гибкой архитектуры. Пример - класс ClickableLabel, расширяющий QLabel.

Работа с БД

Централизованный доступ через класс Database, параметризованные запросы для безопасности, динамическое создание таблиц SQLite.

UI компоненты

Динамическое создание элементов интерфейса, использование системы макетов Qt для адаптивности, компонентный подход.

Обработка данных

Использование QVector и QStringList для хранения данных, паттерн "Модель-Представление", форматирование и валидация данных.

Генерация документов

Низкоуровневая работа с ZIP-форматом для создания DOCX-файлов без сторонних библиотек, управление метаданными файлов.

Примеры реализации

Динамическое создание таблиц SQLite

bool Database::openDatabase(const QString &dbName) {
  db = QSqlDatabase::addDatabase("QSQLITE");
  db.setDatabaseName(dbName);
  if (!db.open()) return false;

  QSqlQuery query(db);
  QString createTableSI =
    "CREATE TABLE IF NOT EXISTS SI ("
    "id INTEGER PRIMARY KEY AUTOINCREMENT,"
    "inv TEXT, naim TEXT, mod TEXT, ...)";
  if (!query.exec(createTableSI)) return false;
  return true;
}

Параметризованные запросы

void Database::insertSI(QString &inv, QString &naim, ...) {
  QSqlQuery query(db);
  query.prepare("INSERT INTO SI (inv, naim, ...) "
    "VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)");
  query.addBindValue(inv);
  query.addBindValue(naim);
  // ... другие параметры
  if (!query.exec()) {
    qDebug() << "Ошибка:" << query.lastError().text();
  }
}

Создание DOCX-файлов

quint32 write_local_file(QDataStream &zip,
  const QString &name, const QByteArray &content) {
  quint32 offset = zip.device()->pos();
  zip << quint32(0x04034b50); // Сигнатура заголовка
  zip << quint16(0x0014); // Версия
  // ... другие параметры ZIP
  zip.writeRawData(name.toLatin1().constData(),
    name.length());
  zip.writeRawData(content.constData(),
    content.size());
  return offset;
}
